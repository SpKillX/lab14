## Архитектура конвейера
`Go-сборщик (Эмуляция API) -> JSON Lines -> Python (Polars) -> Parquet -> DuckDB -> Визуализация (Matplotlib)`

## Описание компонентов
1. **Extract (Go)**: Горутины асинхронно генерируют поток данных ДТП. Реализован канал с буфером на 100 элементов. Пакетная запись (батчами по 50 записей или каждые 2 секунды) сбрасывает данные в `accidents_data.json`. Реализован Graceful Shutdown (перехват `SIGINT`).
2. **Transform (Python/Polars)**: Скрипт читает JSON, очищает от дубликатов, типизирует поля и вычисляет агрегаты (SUM, AVG, MIN, MAX, COUNT). Очищенный датафрейм сохраняется в `accidents_cleaned.parquet`.
3. **Load / Analyze (DuckDB)**: Выполнение SQL-запроса к Parquet-файлу с аналогичными группировками.
4. **Визуализация**: Построение столбчатой и круговой диаграмм распределения инцидентов.
